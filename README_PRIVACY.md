# README — приватность и развёртывание (кратко)

Цель: предоставить максимально приватный анонимный доступ через Tor Hidden Service (.onion).
Ключевые принципы:
- Сервер не сохраняет содержимое сообщений (stateless).
- Backend не логирует тело запросов (в коде мы не пишем такие логи).
- Контейнер слушает только на localhost; Tor проксирует .onion → локалхост:8080.
- Используйте VPS, создавайте отдельный аккаунт для развертывания, удаляйте историю команд после установки, используйте ephemeral VM при необходимости.

Ограничения:
- Хост‑провайдер (VPS) имеет метаданные/логи, которые он может предоставить по запросу.
- Tor скрывает IP сервера от посетителей, но провайдер VPS всё равно видит ваш сервер.
- Для наибольшей приватности используйте провайдера, принимающего криптовалюту, одноразовый email, и создавайте отдельный аккаунт.

Инструкции по использованию:
1. Положите файл модели (например, ggml-gpt4all-*.bin) в ./api/models/model.bin
2. Запустите deploy_vps.sh на VPS (скрипт автоматизирует остальное).
3. После старта скрипт выдаст .onion адрес (путь: /var/lib/tor/hidden_service/hostname).
4. Откройте .onion в Tor Browser на телефоне (или через Orbot + браузер).
